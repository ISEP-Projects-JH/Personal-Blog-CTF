Anatomy of Dependency Failure

Exploring CVE-2019-12989 and the dangers of insufficient sanitization at the driver level. This environment simulates how a secure application logic can still be compromised if the underlying dependency (like libmysqlclient or our fake_MySqldb) fails to guarantee strict ABI compatibility or proper escaping.

Reference: Dependency-Level Vulnerability Model

Why Databases Become Unsafe

The Illusion of Safety

Developers often trust that `mysqlclient` handles all sanitization. However, if the Python wrapper and the C library versions mismatch, protection fails.

Dependency Isolation

To ensure reproducibility, this CTF uses fake_MySqldb. It mimics the behavior where input flows directly into the query builder.

The Attack Vector

Input strings bypassing the sanitization layer allow attackers to break out of SQL quotes, altering the query logic (e.g., ' OR 1=1 --).
